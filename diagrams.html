3. Class & Module Diagram

Lessons 2 & 4

The key types and their relationships (UML-style).

classDiagram
    class FastMCP {
        +name: str
        +instructions: str
        +tool(fn) decorator
        +resource(uri) decorator
        +prompt(fn) decorator
        +run() void
    }

    class SnapTrade {
        +consumer_key: str
        +client_id: str
        +account_information: AccountAPI
        +authentication: AuthAPI
        +reference_data: ReferenceAPI
        +transactions_and_reporting: TxAPI
    }

    class AccountAPI {
        +list_user_accounts(user_id, user_secret)
        +get_user_account_balance(account_id, user_id, user_secret)
        +get_user_account_positions(account_id, user_id, user_secret)
        +get_user_account_orders(account_id, user_id, user_secret)
    }

    class AuthAPI {
        +register_snap_trade_user(body)
        +login_snap_trade_user(user_id, user_secret)
    }

    class ReferenceAPI {
        +symbol_search_user_account(user_id, user_secret, body)
        +list_all_brokerages()
    }

    class TxAPI {
        +get_activities(user_id, user_secret, account_id)
    }

    class ConfigFile {
        +user_id: str
        +user_secret: str
        +path: ~/.snaptrade/config.json
    }

    class EnvVars {
        +SNAPTRADE_CLIENT_ID: str
        +SNAPTRADE_CONSUMER_KEY: str
    }

    FastMCP --> Tool : registers 1 to 10
    FastMCP --> Resource : registers 1 to 2
    FastMCP --> Prompt : registers 1 to 2
    SnapTrade *-- AccountAPI
    SnapTrade *-- AuthAPI
    SnapTrade *-- ReferenceAPI
    SnapTrade *-- TxAPI
    Tool ..> SnapTrade : uses
    Tool ..> ConfigFile : reads
    SnapTrade ..> EnvVars : initialized from

5. Sequence Diagram — Tool Call

Lesson 4

What happens when an AI agent calls any read tool (e.g. get_positions).

sequenceDiagram
    participant AI as AI Client
    participant MCP as FastMCP Server
    participant ENV as Env Vars
    participant CFG as ~/.snaptrade/config.json
    participant SDK as SnapTrade SDK
    participant API as SnapTrade API

    AI->>MCP: tool_call("snaptrade_get_positions", {account_id})
    MCP->>ENV: Read CLIENT_ID, CONSUMER_KEY
    ENV-->>MCP: credentials
    MCP->>MCP: SnapTrade(consumer_key, client_id)
    MCP->>CFG: Read user_id, user_secret
    CFG-->>MCP: user credentials
    MCP->>SDK: get_user_account_positions(account_id, user_id, user_secret)
    SDK->>API: GET /accounts/{id}/positions
    API-->>SDK: HTTP 200 + JSON
    SDK-->>MCP: Response object
    MCP->>MCP: _serialize(response)
    MCP->>MCP: _format_response() → JSON string
    MCP-->>AI: {"account_id": "...", "positions": [...]}

6. Sequence Diagram — First-Time Setup

Lesson 3

The snaptrade_setup tool — user registration and brokerage authorization.

sequenceDiagram
    participant AI as AI Client
    participant MCP as MCP Server
    participant SDK as SnapTrade SDK
    participant API as SnapTrade API
    participant FS as File System
    participant BR as Browser

    AI->>MCP: tool_call("snaptrade_setup")
    MCP->>MCP: _get_client() from ENV

    alt Config exists
        MCP->>FS: Read ~/.snaptrade/config.json
        FS-->>MCP: {user_id, user_secret}
    else First time
        MCP->>MCP: Generate mcp-{uuid} user_id
        MCP->>SDK: register_snap_trade_user({userId})
        SDK->>API: POST /snapTrade/registerUser
        API-->>SDK: {userId, userSecret}
        SDK-->>MCP: registration response
        MCP->>FS: Write config.json (chmod 600)
    end

    MCP->>SDK: login_snap_trade_user(user_id, user_secret)
    SDK->>API: POST /snapTrade/login
    API-->>SDK: {redirectURI}
    SDK-->>MCP: login response
    MCP->>BR: webbrowser.open(redirectURI)
    MCP-->>AI: {"status": "opened", "message": "..."}

    Note over BR: User authorizes
brokerage access

7. Sequence Diagram — Portfolio Summary

Lesson 4

The most complex tool — aggregates data across all accounts with graceful error handling.

sequenceDiagram
    participant AI as AI Client
    participant MCP as MCP Server
    participant SDK as SnapTrade SDK

    AI->>MCP: tool_call("snaptrade_portfolio_summary")
    MCP->>SDK: list_user_accounts()
    SDK-->>MCP: [Account A, Account B]

    loop For each account
        MCP->>SDK: get_user_account_balance(account_id)
        alt Success
            SDK-->>MCP: balance data
        else Error
            SDK-->>MCP: exception
            MCP->>MCP: _clean_error() → store error msg
        end

        MCP->>SDK: get_user_account_positions(account_id)
        alt Success
            SDK-->>MCP: positions data
        else Error
            SDK-->>MCP: exception
            MCP->>MCP: _clean_error() → store error msg
        end
    end

    MCP->>MCP: Assemble portfolio array
    MCP->>MCP: _format_response()
    MCP-->>AI: {"portfolio": [...], "account_count": 2}

9. Serialization Pipeline

Lesson 5

How _serialize() recursively converts SDK objects to JSON-friendly dicts.

flowchart TD
    IN["Input: SDK Response Object"] --> A{"Has .body?"}
    A -- Yes --> B["Unwrap: obj = obj.body"]
    B --> C
    A -- No --> C{"Is list?"}
    C -- Yes --> D["Recurse: [_serialize(item) for item in obj]"]
    C -- No --> E{"Has .to_dict()?"}
    E -- Yes --> F["Call obj.to_dict()"]
    E -- No --> G{"Has __dict__?"}
    G -- Yes --> H["Extract: {k:v for k,v
in __dict__
if not k.startswith('_')}"]
    G -- No --> I["Return obj as-is
(primitive type)"]

    F --> J["_format_response()"]
    H --> J
    I --> J
    D --> J
    J --> K["json.dumps(data,
indent=2,
default=str)"]
    K --> OUT["JSON String
→ AI Client"]

    style IN fill:#ffedd5,color:#7c2d12,stroke:#ea580c,stroke-width:2px
    style OUT fill:#dcfce7,color:#14532d,stroke:#16a34a,stroke-width:2px

12. Server State Diagram

Lesson 7

The lifecycle states of the MCP server and user configuration.

stateDiagram-v2
    [*] --> Unconfigured: Server starts

    Unconfigured --> NoEnvVars: ENV vars missing
    Unconfigured --> NoUserConfig: ENV vars set,
no config.json

    NoEnvVars --> Unconfigured: User sets ENV vars

    NoUserConfig --> Registering: snaptrade_setup called
    Registering --> UserRegistered: API returns user_secret
    UserRegistered --> ConfigSaved: Write config.json
(chmod 600)
    ConfigSaved --> AuthorizingBrokerage: Open browser
redirect URL

    AuthorizingBrokerage --> Ready: User authorizes
brokerage
    AuthorizingBrokerage --> ConfigSaved: User cancels

    Ready --> ServingTools: Any tool called
    ServingTools --> Ready: Response returned
    ServingTools --> ErrorState: API error
    ErrorState --> Ready: Next call succeeds

    Ready --> [*]: Server process exits
